### Scrap.py
O código scrap.py implementa um processo de web scraping para realizar o download automático dos Protocolos Clínicos e Diretrizes Terapêuticas (PCDT) disponibilizados pelo Ministério da Saúde (Não foi possível fazer o scraping de bulas do site da Anvisa, tendo em vista que no FAQ da Anvisa diz que não é permitido o uso de ferramentas automatizadas para baixar documentos). O objetivo do script é coletar os documentos oficiais em formato PDF e armazená-los localmente para que possam ser utilizados posteriormente na etapa de ingestão de dados de uma aplicação RAG (Retrieval-Augmented Generation). 

Após as importações, são definidas a URL principal do portal de PCDTs e a pasta onde os arquivos PDF serão armazenados. O script verifica se o diretório de destino existe e, caso não exista, cria automaticamente a estrutura necessária. Em seguida, é criada uma sessão HTTP reutilizável utilizando requests.Session(), permitindo reaproveitar conexões durante várias requisições. Também é configurado um cabeçalho User-Agent, simulando o acesso de um navegador comum.

A primeira etapa do processo consiste em acessar a página inicial dos Protocolos Clínicos e Diretrizes Terapêuticas do Ministério da Saúde. O HTML retornado pelo servidor é analisado pelo BeautifulSoup, que identifica os links das páginas organizadas por letras do alfabeto. Cada letra representa uma categoria contendo protocolos relacionados a doenças ou condições clínicas específicas. Os endereços dessas páginas são armazenados em uma lista para serem processados posteriormente. Na segunda etapa, o código percorre cada uma dessas páginas de letras e extrai os protocolos disponíveis. Para cada página acessada, o scraper procura pelos elementos HTML que representam os links dos protocolos, coletando o nome da doença e o endereço da página correspondente. Essas informações são armazenadas em uma lista chamada diseases, que contém todos os protocolos encontrados no site.

Depois de reunir a lista completa de protocolos, o script inicia a etapa de localização e download dos arquivos PDF. Para cada protocolo encontrado, o código acessa sua página individual e analisa novamente o HTML em busca do link responsável pelo download do arquivo. O sistema identifica os links que possuem o padrão @@download/file, utilizado pelo portal do Ministério da Saúde para disponibilizar documentos. Quando esse link é encontrado, o endereço do PDF e seu nome são armazenados para iniciar o download. Antes de baixar um arquivo, o código verifica se o PDF já existe no diretório de destino. Caso o documento já esteja presente, o download é ignorado, evitando baixar arquivos duplicados e permitindo que o script seja executado novamente sem repetir todo o processo. Caso o arquivo ainda não exista, o programa realiza o download utilizando requisições em modo streaming, permitindo que o PDF seja salvo em partes menores e evitando o consumo excessivo de memória.

Para aumentar a confiabilidade do processo, o download possui um mecanismo de tentativas. Caso ocorra alguma falha de conexão, o script tenta realizar o download novamente até três vezes, aguardando alguns segundos entre cada tentativa. Além disso, todas as principais operações possuem tratamento de exceções, garantindo que um erro em um protocolo específico não interrompa a execução completa do programa. Ao final da execução, todos os PDFs encontrados no portal do Ministério da Saúde são armazenados no diretório definido. Esses documentos representam a base de conhecimento utilizada posteriormente pelo pipeline de ingestão, onde os arquivos podem ser processados para extração de texto.

### Ingest.py
O código ingest.py implementa uma etapa de ingestão e preparação de documentos PDF para um pipeline de processamento de dados utilizado em uma aplicação RAG (Retrieval-Augmented Generation). Sua função principal é ler os PDFs coletados anteriormente, extrair o texto de cada página, realizar uma limpeza do conteúdo, remover seções consideradas irrelevantes e gerar um arquivo no formato JSONL, que posteriormente será utilizado na etapa de chunking e indexação.

Primeiramente são importadas as bibliotecas necessárias para o processamento dos documentos. O código inicia definindo a configuração dos datasets que serão processados. A variável DATASETS contém informações sobre cada conjunto de documentos, incluindo o tipo do documento, o diretório onde estão armazenados os PDFs de entrada e o caminho do arquivo JSONL que será gerado como saída. Essa estrutura foi pensada para que o mesmo script seja utilizado para diferentes fontes de dados, bastando adicionar novos datasets à lista. No caso apresentado, o dataset configurado corresponde aos Protocolos Clínicos e Diretrizes Terapêuticas (PCDT). Os arquivos PDF são buscados no diretório de dados brutos e o resultado da ingestão é armazenado em um arquivo chamado pcdt.jsonl. 

A função principal do processo é a ingest_directory(). Essa função recebe como parâmetros o diretório contendo os PDFs, o arquivo de saída e o tipo do documento que está sendo processado. Cada PDF encontrado é processado individualmente. Para cada arquivo, o PyPDFLoader realiza a leitura do documento e transforma cada página em um objeto contendo o texto extraído e informações de metadados, como o número da página. Caso ocorra algum erro durante a leitura de um PDF, o código captura a exceção e continua o processamento dos demais arquivos, evitando que um documento corrompido interrompa toda a ingestão. A função normalize_text() é responsável por realizar a limpeza inicial do texto extraído. Como PDFs frequentemente possuem problemas de formatação, como espaços duplicados, quebras de linha excessivas ou caracteres inválidos, essa etapa melhora a qualidade do conteúdo antes do processamento posterior. Depois da normalização, o documento passa pela função remove_references(), responsável por remover a seção de referências bibliográficas (em poucos casos as referências não são identificadas normalmente). Essa etapa é realizada porque referências geralmente possuem citações e informações bibliográficas que possuem pouca utilidade para uma aplicação de busca semântica, podendo prejudicar a recuperação de informações relevantes. Após a limpeza do texto, cada página é transformada em um registro individual no arquivo JSONL. Cada registro contém:

text: o conteúdo textual da página.
source: o nome do arquivo PDF de origem.
page: o número da página.
type: o tipo do documento, neste caso PCDT.

Por fim, o código percorre todos os datasets definidos na variável DATASETS e executa a função ingest_directory() para cada um deles. Essa abordagem torna o pipeline extensível, permitindo adicionar novas fontes de documentos sem modificar a lógica principal do processamento.

### Competências
A competência de construção é demonstrada pelo desenvolvimento de um pipeline automatizado de ingestão de documentos. O código foi responsável por criar uma solução capaz de localizar arquivos PDF, extrair o texto página por página, normalizar o conteúdo, remover informações irrelevantes (como referências bibliográficas) e transformar os documentos em um formato estruturado (JSONL) adequado para as etapas seguintes do sistema. Além disso, a implementação considerou aspectos importantes de engenharia, como tratamento de erros, suporte a múltiplos datasets, preservação de metadados (fonte e número da página) e organização dos dados para permitir rastreabilidade durante a recuperação de informações. A competência de avaliação é demonstrada pela análise da qualidade dos dados gerados e pelo ajuste das estratégias de processamento conforme os problemas encontrados.

### Melhorias
Caso houvesse mais tempo, focaria em rever a implementação de remove_references que apesar de funcionar na maioria das vezes, ainda falha em retirar por volta de 13 seções de referências dos 175 documentos processados e adicionaria a funcionalidade de ver quais documentos já foram processados pelo ingest.py, reduzindo o tempo que leva para gerar o JSONL, tendo em vista que não há necessidade de reprocessar todos os documentos.
